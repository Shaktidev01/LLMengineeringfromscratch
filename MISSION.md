# Mission: Build an LLM — From Scratch to Product

## Why
I want to deeply understand how LLMs work by building one from scratch (Karpathy style), then use that understanding to build a real LLM-powered product or startup. No CS background — learn everything through building.

## Success looks like
- Understand what a transformer is and how training works (not just surface level)
- Build a small working GPT in Python/PyTorch from scratch — trains on my GPU
- Run and query LLMs locally (Ollama, 7B models on RTX 5060)
- Build a real product that uses an LLM as its engine
- Ship something usable to real users

## The path (in order)
1. Mental model — what LLMs are (done ✓)
2. Python fundamentals — just enough for PyTorch (numbers, lists, functions, loops)
3. Math intuition — matrices and vectors, no exams, just feel
4. PyTorch basics — tensors, gradients, training loops
5. Build a tiny GPT from scratch — character-level, Karpathy style
6. Run real LLMs locally — Ollama, 7B models
7. Build a product — wrap an LLM in a real app
8. Fine-tuning — specialise a model on own data
→ Then continue through all 20 phases of AI Engineering from Scratch (aiengineeringfromscratch.com)

## Constraints
- Zero CS/programming background — build everything up from zero
- RTX 5060 8GB VRAM (can train tiny models, run 7B locally)
- Learn by building, not by reading textbooks
- Wants depth AND application — no shortcuts on understanding
- Documenting journey on Twitter — each lesson should produce a tweetable insight

## Success also looks like
- Public Twitter thread documenting the full journey from zero to building LLMs

## Out of scope
- Training GPT-4 scale models (requires thousands of GPUs)
- Pure academic study with no building
